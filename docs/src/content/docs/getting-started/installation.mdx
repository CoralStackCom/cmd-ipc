cmd-ipc ships as both a TypeScript package and a Rust crate. Both speak the same wire protocol.

## Requirements

<Tabs syncKey="lang">
  <TabItem label="TypeScript" icon="seti:typescript">
    - **Node.js** 20.18.2 or higher
    - **TypeScript** 5.0 or higher (recommended)
  </TabItem>
  <TabItem label="Rust" icon="seti:rust">
    - **Rust** 1.75 or higher (edition 2021)
    - An async executor of your choice (tokio, async-std, smol, `futures::executor`, etc.) — the crate is runtime-agnostic
  </TabItem>
</Tabs>

## Package Installation

<Tabs syncKey="lang">
  <TabItem label="TypeScript" icon="seti:typescript">
    <Tabs>
      <TabItem label="npm">
        ```bash
        npm install @coralstack/cmd-ipc valibot
        ```
      </TabItem>
      <TabItem label="yarn">
        ```bash
        yarn add @coralstack/cmd-ipc valibot
        ```
      </TabItem>
      <TabItem label="pnpm">
        ```bash
        pnpm add @coralstack/cmd-ipc valibot
        ```
      </TabItem>
      <TabItem label="bun">
        ```bash
        bun add @coralstack/cmd-ipc valibot
        ```
      </TabItem>
    </Tabs>

    <Aside>
      <a href="https://valibot.dev/" target="_blank" rel="noopener">Valibot</a> is a peer dependency used for schema definition and validation. It's a lightweight alternative to Zod with excellent tree-shaking support. Unlike Zod, Valibot doesn't use `eval` or `new Function()`, making it compatible with strict Content Security Policy (CSP) environments and runtimes like Cloudflare Workers where these are restricted.
    </Aside>
  </TabItem>
  <TabItem label="Rust" icon="seti:rust">
    ```bash
    cargo add coralstack-cmd-ipc
    ```

    Or in `Cargo.toml`:

    ```toml
    [dependencies]
    coralstack-cmd-ipc = "0.1"
    schemars = "1"
    serde = { version = "1", features = ["derive"] }
    serde_json = "1"
    ```

    The core crate re-exports the `#[command]` / `#[commands]` attribute macros from `coralstack-cmd-ipc-macros`, so you don't need to depend on the macros crate directly.

    <Aside>
      <a href="https://docs.rs/schemars" target="_blank" rel="noopener">schemars</a> generates JSON Schema from your Rust request types — the same schema the TypeScript implementation advertises via Valibot. This keeps the wire protocol byte-identical across languages.
    </Aside>

    ### MCP adapter (optional)

    ```bash
    cargo add coralstack-cmd-ipc-mcp
    ```

    Adds `McpServerChannel` for exposing a registry as an MCP server via stdio.
  </TabItem>
</Tabs>

## Language Setup

<Tabs syncKey="lang">
  <TabItem label="TypeScript" icon="seti:typescript">
    ### TypeScript Configuration

    cmd-ipc uses decorators for the `@Command` syntax. Enable decorators in your `tsconfig.json`:

    ```json
    {
      "compilerOptions": {
        "experimentalDecorators": true,
        "emitDecoratorMetadata": true
      }
    }
    ```

    <Aside type="caution">
      Both `experimentalDecorators` and `emitDecoratorMetadata` are required for the `@Command` decorator to work correctly.
    </Aside>

    ### Reflect Metadata

    The decorator system requires `reflect-metadata`. Import it once at your application's entry point:

    ```typescript
    // main.ts or index.ts
    import 'reflect-metadata'

    // Rest of your application
    import { CommandRegistry } from '@coralstack/cmd-ipc'
    ```
  </TabItem>
  <TabItem label="Rust" icon="seti:rust">
    ### Prelude

    Import the common types in one line:

    ```rust
    use coralstack_cmd_ipc::prelude::*;
    ```

    This brings into scope: `command`, `commands`, `ChannelError`, `Command`, `CommandChannel`, `CommandDef`, `CommandError`, `CommandRegistry`, `CommandSchema`, `Config`, `InMemoryChannel`.

    ### Shorter import path (optional)

    If the `coralstack_` prefix feels long at use sites, alias it in `Cargo.toml`:

    ```toml
    [dependencies]
    cmd-ipc = { package = "coralstack-cmd-ipc", version = "0.1" }
    ```

    Then `use cmd_ipc::prelude::*;` works throughout your crate.
  </TabItem>
</Tabs>

## CLI Installation (TypeScript only, optional)

cmd-ipc includes a CLI for generating schemas from remote services. It's included in the main TypeScript package:

```bash
# Generate TypeScript schema from a remote HTTP endpoint
npx cmd-ipc generate-schema --url http://localhost:8787 --output ./src/api-schema.ts
```

Or add it as a script in your `package.json`:

```json
{
  "scripts": {
    "generate-schema": "cmd-ipc generate-schema --url http://localhost:8787 --output ./src/generated/api-schema.ts --prefix api"
  }
}
```

<Aside type="note">
  Rust doesn't yet ship a schema-generation CLI. For now, consume schemas from an HTTP endpoint using the TypeScript CLI and translate by hand, or let the registry's runtime command discovery populate remote command metadata directly.
</Aside>
